# 飞书群消息每日日报

**CRITICAL — 开始前 MUST 先用 Read 工具读取 [`../lark-shared/SKILL.md`](../lark-shared/SKILL.md)，其中包含认证、权限处理**

## 适用场景

- "帮我生成昨天的群日报" / "今天的群消息摘要"
- "/群日报" / "/每日群摘要"
- "看看昨天群里都聊了什么重要的事"
- "把昨天的群消息整理一下发给我"

## 前置条件

仅支持 **user 身份**。执行前确保已授权：

```bash
lark-cli auth login --domain im
```

## 工作流

```
{日期} ─► im +chat-messages-list ──► 各群昨日消息
               │
               ▼
         AI 提炼（关键信息 + 决策 + 待办 + 风险）
               │
               ├─► 飞书私信推送给用户
               ├─► 飞书群推送（可选）
               └─► Obsidian Markdown 存档
```

### Step 1: 确定日期范围

默认 **昨天**。推断规则：
- "昨天" → 昨天 00:00 ~ 23:59
- "今天" → 今天 00:00 ~ 当前时间
- "上周五" → 具体日期

> **注意**：日期计算必须调用 `date` 命令验证，禁止心算。时间格式为 ISO 8601（如 `2026-04-27T00:00:00+08:00`）。

### Step 2: 扫描群消息

对每个监控群执行消息拉取：

```bash
lark-cli im +chat-messages-list \
  --chat-id "<chat_id>" \
  --start-time "<昨天00:00 ISO8601>" \
  --end-time "<昨天23:59 ISO8601>" \
  --page-size 100 \
  --as user
```

**监控群列表（按需调整）：**

| 群名 | chat_id |
|------|---------|
| 美的百商计划 | oc_fe5ea364d34a2d6a0c6a20cbfd8d5211 |
| 美的私域会员B端运营 | oc_530fe1c425b115cd76f909717e7a0a5f |
| 美的企微基础运营小组 | oc_7f9d2bf9e2a5c16bd85f7052ef60b242 |
| 零一数科（全员群） | oc_8bbb0f489195676592120355c564d18d |
| 天骐的工作AI赋能群 | oc_75cd9241e91959f06cf01cf687814365 |

**数据提取规则：**
- 从 JSON 中提取 `msg.sender.name`（发送者）、`msg.body.content`（内容）
- content 为 JSON 字符串，需二次解析取 `text` 字段
- 过滤空消息和机器人消息（sender type 为 bot 的跳过）
- 按时间排序

### Step 3: AI 提炼分析

将所有群消息汇总后，进行以下分析：

1. **关键信息提取**：识别讨论的核心话题（每群最多 5 个）
2. **决策记录**：提取明确的决策或结论（有"决定""确认""同意"等关键词）
3. **待办事项**：提取需要跟进的任务（有"截止""deadline""待办""需要"等关键词）
4. **风险/异常**：识别潜在风险或异常信息（有"风险""问题""注意""紧急"等关键词）
5. **重要通知**：标记正式通知类消息

### Step 5: 推送

**私信推送（默认）：**
```bash
lark-cli im +messages-send \
  --user-id "ou_ca807ea46bb97f94e9adebd03e34a606" \
  --text "<日报内容>" \
  --as user
```

**群推送（可选，用户指定时执行）：**
```bash
lark-cli im +messages-send \
  --chat-id "<目标群chat_id>" \
  --text "<日报内容>" \
  --as user
```

> **注意**：b2b 自建应用发群消息受限，群推送需确保机器人已在目标群中。

**Obsidian 存档：**
```
文件路径：{OBSIDIAN_DIR}/07-主业工作/{当前周目录}/飞书群日报-{YYYY-MM-DD}.md
```
