# Scenario Library

This file provides reusable scenario anchors for the most common triggers.

## Scenario 1: Seeing peers make money

Typical user lines:

- "别人已经开始赚大钱了，我连网站都没上。"
- "同一起跑线的人都出结果了，我感觉自己好差。"

Likely emotional pattern:

- comparison-triggered shame
- urgency
- fear of being left behind

Likely interpretation pattern:

- "别人快 = 我差"
- "我还没做出来 = 我没希望"

Suggested progression:

1. acknowledge how sharp this kind of comparison hit is
2. normalize that it hurts because the user cares
3. separate others' progress from the user's identity
4. name what stage the user is actually in
5. return to concrete bottlenecks and next steps

## Scenario 2: Product not launched, self-worth collapse

Typical user lines:

- "网站还没上线，我是不是根本不行。"
- "产品没出来，我觉得自己很废。"

Likely emotional pattern:

- frustration
- shame
- inner collapse after unfinished execution

Likely interpretation pattern:

- unfinished work -> personal failure

Suggested progression:

1. hold the disappointment
2. separate unfinished step from identity
3. ask what specifically is blocked
4. define the smallest shippable next move

## Scenario 3: No feedback after effort

Typical user lines:

- "做了一段时间没反馈，我开始怀疑自己。"
- "这么久还没人付费，是不是说明我不行。"

Likely emotional pattern:

- discouragement
- doubt
- action fatigue

Likely interpretation pattern:

- no feedback -> no value

Suggested progression:

1. acknowledge how demoralizing ambiguity is
2. reframe feedback absence as information, not identity proof
3. identify whether the issue is offer, channel, message, or patience horizon
4. define one next validation step

## Scenario 4: Panic after realizing the gap

Typical user lines:

- "我现在越看越慌。"
- "我感觉自己被时代甩开了。"

Likely emotional pattern:

- panic
- chaos
- collapse of inner order

Suggested progression:

1. stabilize first
2. reduce urgency
3. get one clean description of what happened
4. only then enter interpretation work

## Scenario 5: Using emotion to avoid the real problem

Typical user lines:

- "我就是很差，所以没什么好说的。"
- "算了，我可能就不是这块料。"

Likely hidden mechanism:

- self-condemnation used as a fast exit from reality contact

Suggested progression:

1. acknowledge pain
2. gently expose the pattern
3. immediately reveal the positive intention
4. ask what concrete problem the user may be bypassing
5. name one minimal next move
