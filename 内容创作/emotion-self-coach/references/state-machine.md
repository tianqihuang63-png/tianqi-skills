# State Machine

## Purpose

This file defines how to judge the user's current state and whether to advance or stay.

The agent should not rigidly march through steps.
It should move only when the user has enough stability.

## State A: Flooded

Typical signals:

- repeated extreme self-condemnation
- panic-heavy or chaotic language
- "我完了" / "我不行了" style collapse
- inability to describe what actually happened

Agent goal:

- stabilize
- reduce immediate self-attack
- avoid interpretation-heavy analysis

Allowed moves:

- emotional acknowledgment
- normalization of being hit
- gentle slowing down
- one small anchoring distinction

Avoid:

- root-cause lectures
- hidden-mechanism exposure
- action planning

## State B: Partially Settled

Typical signals:

- user can describe the trigger
- user can say why it hurts
- intensity is present but language is more coherent

Agent goal:

- separate fact from evaluation
- begin interpretation repair

Allowed moves:

- "发生了什么" vs "你怎么解释它"
- identify self-judgment language
- offer alternative explanation frames

## State C: Reflective Capacity Present

Typical signals:

- user is no longer only venting
- user can receive structure
- user can tolerate being shown a pattern

Agent goal:

- gently expose the mechanism that keeps the user stuck

Allowed moves:

- name self-attack
- name retreat justification
- name how emotion may be covering the real problem

Constraint:

- every exposure move must also reveal positive intention

## State D: Agency Returning

Typical signals:

- user can say what they really want
- user can distinguish comparison from desire
- user is willing to face a concrete bottleneck

Agent goal:

- identify the current real problem
- define one minimal next action

Allowed moves:

- ask what is actually missing
- define this week's smallest progress move
- identify who to ask or what to test

## Transition rules

- Stay longer in A if the user remains flooded.
- Only enter C when the user can tolerate structure without shutting down.
- Only enter D when the user can hold reality without collapsing into self-judgment.
- If the user re-floods, return to stabilization instead of pushing through.
